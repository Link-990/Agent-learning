# Transformer 架构和核心组件

## 🎭 面试官这样问

> 面试官：Transformer 的核心组件有哪些？简单说说。

> 你：就是 Attention、FFN、LayerNorm、残差连接这些，Attention 负责捕捉上下文关系，FFN 做非线性变换……

> 面试官：停停停，你这背的是名词清单，不是理解。我问你，为什么非要 Attention？为什么不能像 RNN 那样串行读？残差和 LayerNorm 到底谁先谁后？为什么顺序错了训练就崩？

> 你：呃……Attention 能并行，所以比 RNN 快。残差是先加再归一化……吧？

> 面试官：你想想，如果一层 Transformer Block 什么都不做，它应该能原封不动把输入传到下一层，这是谁在保证？如果 Attention 算出来的数值暴增暴减，又是谁在兜底？把这些串起来想，不要背零件名字。

> 你：啊……所以残差保证"最差也能原样传递"，LayerNorm 保证"数值别爆炸"，Attention 负责"从别处取信息"，FFN 负责"自己消化信息"，四个组件各管一件事？

> 面试官：方向对了。那位置信息呢？Attention 本身根本不关心顺序，"猫追狗"和"狗追猫"它看到的是一样的关系，怎么办？

> 你：需要位置编码！否则模型分不清谁在前谁在后。

> 面试官：好，现在把整条链路串一遍：一个 token 从进来到变成 logits 输出，经历了什么？

【要把 Transformer 的组件串成一条链路，而不是背散装名词，这才是面试官真正想听的。】

## 💡 简要回答（面试话术）

Transformer 是一种"让序列里所有 token 同时互相看"的架构，核心是一条反复执行的增量更新链路。

你可以把一层 Transformer Block 想象成一个翻译团队里的两轮讨论：第一轮每个人都看一遍别人在说什么，取自己需要的信息（自注意力）；第二轮每个人独自消化刚才收集到的信息，做深度加工（FFN）。这两轮之间各有一条"旁路"保证原始信息不丢失（残差连接），每轮开始前还要把数值标准化一下，防止信息打架（LayerNorm）。位置信息靠额外编码补进去，因为注意力本身对顺序是"脸盲"的。

面试时只要把这个"两轮讨论 + 两条旁路 + 位置补丁"的逻辑讲通，再补上训练推理差异和长上下文成本，Transformer 架构题基本不会翻车。

## 📝 详细解析

### 核心矛盾：全局交互 vs 可训练性

Transformer 要同时解决两个打架的需求。

第一个需求：文本理解需要全局关系。比如"我上周买的耳机，包装还在，但发票丢了，还能退吗？"——"发票丢了"和"还能退吗"之间隔了好几个词，模型必须把限制条件和结论跨距离挂上钩。RNN 时代按时间步一个个读过去，信息传太远就衰减了，就像传话游戏，传十个人之后原话早变味了。

第二个需求：模型要能大规模训练。RNN 必须等上一步算完才能算下一步，GPU 的并行能力使不上劲。Transformer 换了个思路：把整段序列里的 token 同时摆到桌面上，让它们通过自注意力直接建立联系——"耳机""发票""能退"在同一层就能互相对话，不必等信息沿着时间步慢慢爬过来。

这两个需求的平衡点就是 Transformer 的架构设计：用自注意力解决"谁跟谁相关"，用 FFN 解决"每个位置怎么加工"，用残差和 LayerNorm 解决"堆深了怎么别崩"，用位置编码解决"顺序从哪来"。

### 底层机制：一层 Block 的四件套怎么配合

我给你画一条完整的处理链路。假设你已经把"苹果发布了新手机"送进模型：

**第一步：Tokenize + Embedding + 位置信息**

文本先被切成 token，查表变成向量。关键来了——"苹果"这个词的初始 embedding 是固定的，但这句话里它指代公司，下句话里可能指代水果。怎么区分？靠上下文。但在还没看上下文之前，模型得先知道这些 token 的先后顺序，所以要在 embedding 上叠加位置编码。常见的做法有绝对位置编码（sin/cos）、相对位置编码和 RoPE——不管哪种，目的都是告诉模型："苹果"在位置 1，"发布"在位置 2，别搞混。

![Transformer 概念总览](https://raw.githubusercontent.com/Link-990/Agent-learning/main/images/02-llm-framework/01-transformer/01-概念总览.png)

**第二步：进入 Transformer Block（循环多次）**

一层 Block 可以记成两句话：

```text
x = x + self_attention(LayerNorm(x))    # 第一轮：跟别人交流
x = x + feed_forward(LayerNorm(x))      # 第二轮：自己消化
```

注意，这里是 **Pre-Norm** 结构（先归一化再计算），也是现在主流大模型的写法。旧论文里是 Post-Norm（先计算再归一化），但实践证明 Pre-Norm 训深层网络更稳。

第一轮（自注意力）：经过 LayerNorm 稳定数值后，每个 token 生成 Q（我要查什么）、K（我能被什么匹配）、V（我实际提供什么），然后互相查、互相取。比如"苹果"的 Q 去查"发布""财报"的 K，发现高度匹配，那就从它们的 V 里取走"公司属性"的信息。这一步之后，"苹果"的向量就从"苹果=？"变成了"苹果≈公司"。

第二轮（FFN）：每个位置独立做非线性变换。你可以把它理解成"刚才从别人那听了一堆东西，现在自己关起门琢磨琢磨"。FFN 通常是两层全连接加一个激活函数，比如 `W2(GELU(W1(x)))`，中间维度比 hidden size 大好几倍，是一个"膨胀再压缩"的过程，给模型足够的表达能力去消化和重组信息。

**第三步：两层中间的那道"保险"——残差连接**

`x + f(x)` 看起来只是个加法，但它是深网络的命根子。如果没有残差，每一层都完全覆盖旧表示，一旦某一层学偏了，后续层全跟着歪。有了残差，每一层学的是"增量修改"：学好了就加上去，没学好就 ≈ 原样传递。**你可以把残差连接理解成高速公路的应急车道——主路堵了，车还能从应急车道走。** 信息和梯度也能通过这条应急通道直达后面的层，极大缓解了深层网络的梯度消失。

**第四步：LayerNorm——数值交警**

注意力算出来的分数、FFN 的激活值，都可能出现数值暴增暴减。LayerNorm 在每个位置上做标准化，把数值拉回到一个可控的范围，让下一层不会接收到"离谱"的输入。没有它，层数一深，激活分布会越来越偏，训练直接崩掉。

**第五步：从 hidden state 到 logits**

最后一层 Block 输出后，取最后一个位置的 hidden state，通过 `lm_head`（一个线性映射）投影到词表大小，得到 logits——也就是词表里每个 token 作为"下一个 token"的未归一化分数。logits 不是最终答案，还要经过 temperature、top-p、top-k 等解码策略，才变成实际输出的 token。

### 工程例子：为什么同一个词在不同句子里"变了"

这是我面试时最爱举的例子，因为它把"动态表示"这个概念讲活了。

看两句话：
- "这个苹果很好吃。"
- "苹果发布了财报。"

初始时，"苹果"这个 token 的 embedding 向量在这两句话里完全相同，它还没看上下文。但经过多层 Transformer 后，成果截然不同：

第一句话里，"好吃"的 attention 权重高，FFN 把"水果属性"注入"苹果"的表示；第二句话里，"发布""财报"的 key 匹配到了公司语义的 query，FFN 注入了"公司属性"。**这个过程不是查词典查出来的，而是注意力在表示空间里动态聚合出来的。**

面试官追问："那如果一句话里同时有水果和公司含义，比如'苹果从苹果园采购原料'，模型怎么办？"——多头注意力！不同 head 可能分别捕捉到"公司"和"水果"两重语义，然后在 FFN 里做融合。这也是为什么单头注意力不够用。

再看 RAG 场景。用户问题、检索出来的 5 个片段、系统提示词，被拼成一个长 prompt 送进 Transformer。模型需要在生成答案时同时权衡：用户真正在问什么？资料里哪些信息可靠？系统要求我不要编造。理论上注意力能看全局，但上下文太长时噪声也多——模型未必总能把最关键的信息排在最高优先级，这就是长上下文 RAG 仍会答偏的根因。

### 边界和风险：别把结构神化

**第一，自注意力复杂度 ≈ O(n²)。** 序列从 4K 增到 32K，注意力矩阵不是线性变大，而是近似 64 倍。显存、延迟、调度压力都会飙升。能接长输入 ≠ 无成本。

**第二，位置编码是"借"来的顺序感。** 注意力本身对顺序脸盲，全靠位置编码补。如果位置编码错了——比如推理时 position id 和训练时不对齐，或者 prefill 和 decode 阶段拼接错——模型可能把"猫追狗"理解成"狗追猫"。RoPE、ALiBi 等方案都在试图让位置编码更灵活，但没有"万能"方案。

**第三，训练和推理是两个世界。** 训练时可以并行计算整段序列的 next token loss，梯度一次性回传。自回归推理时必须一个 token 一个 token 生成，这是串行的，改不了。KV Cache 只是减少了每步的重复计算，没改变生成步骤本身的串行性。

**第四，注意力权重 ≠ 完整解释。** 某层某个 head 的权重图显示它关注了某个词，不代表模型"因为这个词才这么回答"。最终输出还经过 FFN、多层叠加、残差路径和采样策略。把注意力图当成模型的"思考轨迹"，是面试中常见的过度简化。

## 🎯 面试总结

- **必答 3 点**：① Transformer = 自注意力（全局信息交互）+ FFN（位置独立加工）+ 残差（信息高速公路）+ LayerNorm（数值交警）+ 位置编码（顺序补丁）；② 一层 Block = 两轮增量更新，每轮都是 Pre-Norm → 计算 → 残差加法；③ 训练并行（一次性看全序列算 loss），推理串行（逐 token 自回归）。
- **加分项**：能说出 Pre-Norm vs Post-Norm 的区别和趋势；能解释长上下文 O(n²) 的具体含义和工程影响；能把 RAG 场景下模型"为什么仍会答错"从上下文长度和注意力稀释角度讲清楚。
- **一句话总结**：**Transformer 的本质不是 Attention，而是一套"让 token 反复互相看、反复自己琢磨、同时保证信息不丢、数值不乱"的增量更新流水线。**

## 🔍 排查和实践建议

线上出问题，按四层排查：

**输入层**：tokenizer 是否把中文空格、代码缩进、特殊符号切得异常？prompt 是否被截断？位置编码和模型权重是否匹配？

**配置层**：层数、hidden size、注意力头数是否和权重文件一致？千万别拿 LLaMA-7B 的配置去跑 LLaMA-13B 的权重。

**解码层**：temperature 是不是太高导致胡言乱语？top-p 是不是太宽导致随机性过大？max tokens 和 stop sequence 是否合适？

**性能层**：首 token 延迟（看 prefill 和 prompt 长度）、decode tokens/s（看 KV Cache 和 batch）、显存峰值（看上下文长度和并发）。

面试前建议你能在白板上画出一层 Block 的完整数据流：输入 → LayerNorm → Attention → 残差加 → LayerNorm → FFN → 残差加 → 输出。每画一步，讲清楚它"管什么"和"没它会怎样"。这条链路讲透了，Transformer 的架构题就稳了。
