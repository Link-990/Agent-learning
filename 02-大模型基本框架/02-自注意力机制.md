# 自注意力机制

## 🎭 面试官这样问

> 面试官：自注意力是怎么工作的？用你自己的话说。

> 你：就是输入投影成 Q、K、V，然后 Q 乘 K 的转置得到权重，softmax 之后乘 V，就得到上下文表示。

> 面试官：你这是在背公式。我问你，为什么不直接比较原始的词向量？为什么要拆成 Q、K、V 三个而不是两个？`sqrt(d_k)` 这东西为什么要除？不除会怎样？

> 你：嗯……Q 表示要查什么，K 表示能被匹配的特征，V 表示实际传递的内容。除 `sqrt(d_k)` 是为了防止点积太大导致 softmax 梯度消失……

> 面试官：好，那你再说说，causal mask 和 padding mask 有什么区别？训练时如果把 causal mask 漏了，会发生什么？

> 你：causal mask 让当前 token 不能看未来的 token，padding mask 让模型不关注填充的空位置……漏掉 causal mask 的话，训练时模型偷看到了正确答案，推理时看不到，性能会崩掉。

> 面试官：方向对。那我再问一个——注意力权重能不能直接解释模型的输出？比如某个 head 特别关注了"发票"这个词，我能不能说"模型就是因为看到了发票才这么回答的"？

> 你：不能完全这么说。注意力权重只反映一层里信息聚合的比例，后面还有 FFN、多层叠加和采样，不能简单归因。

> 面试官：好，那最后把多头注意力的完整计算流程串一遍，从输入 X 到最终输出，形状怎么变。

【把自注意力从"背公式"变成"讲道理"，从"QKV 是什么"讲清楚"为什么是 QKV"，这才是面试官想听的深度。】

## 💡 简要回答（面试话术）

自注意力做一件事：让序列里每个 token 更新自己时，去其他 token 那里"取经"。

你可以想象一个微信群聊。群里每个人都在说话（每个 token 是一条消息）。现在轮到小张更新自己的理解——他先想好自己大概想知道什么（Query），然后看每个人的发言标题能不能对上自己的需求（Key 匹配），对得上的，就认真听（高权重），对不上的就略过（低权重），最后把大家说的话按权重合成一个总结（Value 加权）。这就是一轮自注意力。

为什么要拆成 Q、K、V？因为"我想知道什么""你能被什么匹配""你真正传递什么"是三个不同的信息维度。如果只用一个向量同时做这三件事，表达能力严重受限——就像一个人同时要当记者、图书馆检索员和内容提供者，哪个都做不精。多头注意力相当于开多个并行群聊，每个群聊关注不同的话题维度。

## 📝 详细解析

### 核心矛盾：相关性判断和信息聚合

一个 token 要更新自己，必须解决两个问题：**我该关注谁**（相关性判断）和**我从对方那里拿什么**（信息聚合）。这两个问题的输入输出不同，用一个向量同时搞定是强人所难。所以 Transformer 把输入投影到三个不同的表示空间：

- **Query**："我现在想找什么信息"——由当前 token 自己生成，反映它当下的需求。比如在"苹果很好吃"里，位置 2 的 Q 可能在问"主语到底是什么？是水果还是公司？"
- **Key**："我能被别人用什么特征匹配到"——每个 token 生成的"标签"，方便别人检索自己。比如"好吃"这个 token 的 K 会带上"我是食物属性"的特征。
- **Value**："如果别人关注我，我实际提供什么内容"——真正的信息载体。匹配成功后，对方取走的是 V，不是 K。

核心公式：

```text
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) V
```

拆解每一步：
1. **`QK^T`**：得到 `[seq_len, seq_len]` 的分数矩阵。第 i 行第 j 列表示 token i 对 token j 的"关注程度"原始分数。
2. **`/ sqrt(d_k)`**：缩放。因为 Q 和 K 的每个元素方差 ≈ 1，点积后方差 ≈ d_k，数值可能很大，softmax 接近 one-hot，梯度几乎为零。除以 `sqrt(d_k)` 把方差拉回 ≈ 1，让 softmax 保持柔和。
3. **softmax**：沿最后一维（每行）做，把每行的分数变成正数且和为 1 的概率权重。
4. **`× V`**：用权重加权 V，得到上下文聚合后的表示。

### 底层机制：从输入到输出，形状怎么变

假设输入 `X` 是 `[batch, seq_len, hidden_size]`。多头注意力把 `hidden_size` 拆成 `num_heads` 份，每个 head 维度 `head_dim = hidden_size / num_heads`。

完整流程：

```text
Q = X @ Wq    # [batch, seq, hidden] → [batch, seq, hidden]
K = X @ Wk
V = X @ Wv

# 拆成多头：reshape 成 [batch, num_heads, seq, head_dim]
# 每个头独立计算
scores = Q @ K.transpose(-2, -1) / sqrt(head_dim)   # [batch, heads, seq, seq]
scores = scores + mask                                  # 遮住不该看的
weights = softmax(scores, dim=-1)                       # 沿最后一维 softmax
context = weights @ V                                   # [batch, heads, seq, head_dim]

# 拼接多头
output = context.reshape(batch, seq, hidden) @ Wo       # 融合多头输出
```

面试时最容易被深挖的是 **mask**。分为两种：

**Causal Mask（因果遮罩）**：Decoder-Only 模型的命根子。生成第 t 个 token 时，只能看前 t 个位置，不能偷看 t+1 及之后的"未来答案"。做法是把未来位置的分数设成 `-inf`（或一个极大的负数），softmax 后这些位置的权重 ≈ 0。**漏掉 causal mask = 考试时偷看了答案，上了考场才发现没答案可看。**

**Padding Mask（填充遮罩）**：批量推理时，不同句子长度不一样，短句后面会补 `[PAD]` token。padding mask 让模型不关注这些空位，否则空位的随机向量会污染注意力计算。

实践中两种 mask 经常同时出现：先用 padding mask 屏蔽空位，再用 causal mask 屏蔽未来。任何一个处理错，都可能导致困惑度异常、输出不稳定，甚至同一个样本在不同 batch size 下答案不同——这是因为 padding 数量变了，mask 的覆盖范围不一样。

### 工程例子：多头注意力为什么不是"多多益善"

单头注意力的局限很好理解：它只有一个投影空间，所有关系都挤在同一个空间里判断。如果一句话里既有语法依赖、又有实体指代、还有情感倾向，单头只能把它们混在一起，像一个人同时盯五台监控屏幕，很容易顾此失彼。

多头注意力的做法是把 `hidden_size` 拆成多个子空间，每个 head 在自己的子空间里独立计算注意力。**这就好比你派了 8 个分析师去分析同一段会议记录：A 关注谁说了什么，B 关注情绪倾向，C 关注数字和日期，D 关注人称代词——最后把 8 份报告合在一起，信息量远比一个人盯全场大。**

但面试时千万别说什么"第 3 个头负责语法、第 7 个头负责指代"——这种归因是过度简化。更严谨的说法是：多头注意力提供多个子空间，让模型能同时表达多种相关性模式；某些可视化结果可能呈现出人类可解释的模式，但这不是固定分工，而是训练中自发形成的。

看一个具体的 RAG 失败场景。上下文里有两段资料：

- 资料 A："本店支持 15 天无理由退货，商品需保持原包装。"
- 资料 B："发票丢失不影响正常退货权益，但无法开具纸质发票凭证。"

用户问："耳机买了 10 天，发票丢了，还能退吗？"

好的注意力应该让"发票丢了"和资料 B 的限制条款高权重关联，让"10 天"和资料 A 的 15 天期限对得上。但如果 chunk 被切得太碎、prompt 太长导致关键信息被冲淡，注意力可能只捕捉到"退货"这一个信号，漏掉"发票丢了会影响什么"的细节，从而给出过度乐观的答案。**这是自注意力的"信息稀释"问题——看得多不等于看得准。**

### 边界和风险：注意力的五个误区

**误区一：注意力权重就是模型的"思考过程"。** 错。权重只反映某层某个 head 的信息聚合比例。最终输出还经过多层叠加、FFN 的非线性变换、残差路径的混合和采样策略的选择。一个 head 关注了某个词，不代表整个模型"因为这个词才这么回答"。

**误区二：O(n²) 只是理论问题。** 不是。序列从 4K 翻到 32K，注意力矩阵从 16M 涨到 ≈ 1B 个元素，显存、访存带宽、计算延迟都有真实的工程代价。长上下文模型能处理 128K token，不代表你可以无脑塞满。

**误区三：向量相似 = 业务相关。** 自注意力在 prompt 内部聚合信息，向量检索在知识库做相似度匹配——两者都可能把"语义相近但不能回答问题"的内容当成有效信息。RAG 里常需要 reranker 做二次筛选，就是这个原因。

**误区四：mask 错了也能凑合用。** 不能。causal mask 漏掉 = 训练推理不一致；padding mask 位置处理错 = batch 敏感；position id 不连续 = 位置编码混乱。这些问题不会报错，但会让模型输出变得"微妙地不对"。

**误区五：`sqrt(d_k)` 是可有可无的细节。** 没有它，大维度下 softmax 直接变成 argmax，梯度几乎为零，模型学不动。这是数学上的硬伤，不是调参能绕过去的。

## 🎯 面试总结

- **必答 3 点**：① Q 查需求、K 给标签、V 传内容，三者分工让注意力既灵活又强大；② `QK^T / sqrt(d_k) → softmax → × V` 这条流水线的每步都要讲出"为什么"；③ causal mask 防偷看未来，padding mask 防空位污染，两者经常叠加使用。
- **加分项**：能说出多头注意力不是固定分工而是子空间并行；能讲清长序列 O(n²) 的工程含义（不只是复杂度符号）；能区分自注意力（prompt 内聚合）和向量检索（知识库匹配）的异同。
- **一句话总结**：**自注意力是一套"每个 token 用自己的需求去匹配别人的标签，按匹配程度取走别人的内容"的动态信息聚合机制，多头和多层让这套机制能同时捕捉多种关系模式。**

## 🔍 排查和实践建议

**训练排查**：loss 不降或异常波动，先查标签右移是否正确（预测第 t 个 token 时，输入是前 t-1 个），padding 位置的 loss 是否被 mask 掉，causal mask 是否正确遮住了未来 token。

**推理排查**：同一个 prompt 在不同 batch size 下输出不一致？重点查 padding mask 是否按实际 padding 数量动态生成，position id 是否对齐，KV Cache 拼接是否正确。

**学习建议**：拿 4 个 token、head_dim=8 手算一遍形状——`Q:[4,8] × K^T:[8,4] → scores:[4,4] → softmax → [4,4] × V:[4,8] → context:[4,8]`。把形状、mask、缩放和多头拼接全部走通，自注意力就不再是"好像懂"而是"真懂"。
